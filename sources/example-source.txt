Evidence Traceability In Review Workflows

Evidence traceability links each user claim to the source material that supports, weakens, or contradicts it. A reviewer can inspect the matched excerpts before accepting a recommendation, which improves review quality and reduces unsupported decisions.

In an EvidencePilot-style workflow, uploaded source documents are extracted into plain text, split into smaller chunks, and stored with source metadata. When a user submits a claim, the system retrieves the most relevant chunks first instead of sending the whole document to the local model.

The local AI model should be used after retrieval to explain why a candidate excerpt is suitable. This keeps the context focused, makes the answer easier to audit, and avoids relying on the model to read long PDF or DOCX files directly.

For scanned or complex PDF documents, MinerU should run before chunking. MinerU converts the document into cleaner Markdown text, preserving headings, tables, and reading order better than a basic text extractor.
